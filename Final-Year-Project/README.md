## 📋 FYP Execution Plan

### Step 1: Topic Selection
- [ ] Shortlist 3 topics
- [ ] Check with guide/professor for approval
- [ ] Research existing similar systems
- [ ] Define what makes yours unique

### Step 2: Planning & Design (Week 1-2)
- [ ] Define all features (scope document)
- [ ] Design database schema (ER diagram)
- [ ] Create wireframes for all pages
- [ ] Plan user roles and permissions
- [ ] Set up GitHub repo

### Step 3: Database Design
- [ ] Identify all entities and relationships
- [ ] Normalize tables (at least 3NF)
- [ ] Define primary & foreign keys
- [ ] Create MySQL database and tables
- [ ] Insert sample/dummy data for testing

### Step 4: Development (Core → Features → Polish)
- [ ] Setup project folder structure (MVC)
- [ ] Build authentication system first
- [ ] Build core features one by one
- [ ] Add admin panel
- [ ] Implement security measures
- [ ] Test each feature thoroughly

### Step 5: Documentation
- [ ] Project synopsis (1-2 pages)
- [ ] ER diagram
- [ ] Data flow diagram (DFD)
- [ ] Screenshots of all pages
- [ ] User manual
- [ ] Technical documentation

### Step 6: Presentation Prep
- [ ] Prepare demo with test data
- [ ] Prepare slide deck
- [ ] Practice live demo flow
- [ ] Prepare answers for common questions
- [ ] Deploy on local server for demo

## 💬 Common Viva Questions

- Why did you choose this project?
- What technologies did you use and why?
- How did you handle security?
- What are the limitations of your project?
- What would you add in the future?
- Explain your database design
- How does authentication work in your project?
- What is SQL injection and how did you prevent it?
- What is the difference between GET and POST?
- How do sessions work in PHP?

## ✅ Final Checklist Before Submission

- [ ] All features working without errors
- [ ] No hardcoded credentials in code
- [ ] All forms validated (frontend + backend)
- [ ] Security implemented (SQL injection, XSS, CSRF)
- [ ] Responsive design (works on mobile)
- [ ] Database backed up
- [ ] Code pushed to GitHub
- [ ] Documentation complete
- [ ] Demo data inserted
- [ ] Tested on clean browser (no cache)
